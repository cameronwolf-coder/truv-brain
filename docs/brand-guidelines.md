## Visual Identity

### Color Palette

| Color | Hex Code | Usage |
|-------|----------|-------|
| **Truv Blue (Primary)** | `#2C64E3` | CTAs, buttons, links, key highlights |
| **Accent Blue** | `#2A5FD8` | Secondary buttons, hover states |
| **Background** | `#F3F3F3` | Page backgrounds |
| **Dark** | `#171717` | Primary text, headings |
| **Gray** | `#3C3C43` | Secondary text, body copy |
| **White** | `#FFFFFF` | Cards, inputs, containers |
| **Border Gray** | `#A7A7A7` | Input borders |

### Usage Guidelines
- **Truv Blue** is the primary brand color. Use for CTAs, links, and key UI elements.
- **Dark** for all primary text and headings.
- **Gray** for secondary text and supporting content.
- Maintain high contrast for accessibility.

### Typography

**Primary Font Family:** Gilroy
- Clean, modern sans-serif
- Used for all headlines and body text
- Fallback stack: `Gilroy, Inter, sans-serif`

**Font Weights:**
- **500 (Medium)** - Standard body text and UI elements
- **600-700** - Headings and emphasis

**Font Sizes (Web):**
- H1: 87px (hero headlines) - scales down on mobile
- H2: 24px (section headers)
- Body: 15px (paragraph text)

### Design Elements
- **Border Radius:** 20px (cards, containers)
- **Input Border Radius:** 50px (pill-shaped inputs)
- **Color Scheme:** Light mode default
- **Framework:** Astro v5 + Bootstrap
- **Base Unit:** 4px spacing system

### Logo

**File Location:** `/assets/images/logo/logo-truv.svg`

**Usage Rules:**
- Use SVG format for web
- Maintain clear space around logo
- Don't stretch, rotate, or modify proportions
- Preferred placement: top-left header
- Use white/dark variants based on background

## Voice & Tone

### Brand Voice Characteristics

#### 1. Confident & Authoritative
- Position as industry leader
- Use data and proof points
- Examples:
  - "The only unified platform purpose-built for..."
  - "Industry-leading conversion rates"
  - "Approved by Fannie Mae and Freddie Mac"

#### 2. Clear & Direct
- Plain language, not jargon-heavy
- Short, punchy sentences
- Active voice preferred
- Examples:
  - "Make confident decisions."
  - "Verify borrowers instantly."
  - "Save 80% on verifications."

#### 3. Professional but Approachable
- B2B tone with warmth
- Educational without condescension
- Examples:
  - "We'd love to set up a demo"
  - "Let's chat about..."
  - "Reach out to learn more"

#### 4. Customer-Centric
- Focus on benefits, not features
- Emphasize ROI and cost savings
- Examples:
  - "Delight your users with a seamless experience"
  - "Save $350 or more per closed loan"

### Writing Style Guidelines

**Headlines:**
- Use action verbs: "Unlock," "Accelerate," "Reduce," "Verify"
- Lead with benefits
- Keep under 10 words for hero headlines
- Examples:
  - "Mortgage income and employment verification in minutes."
  - "Automated income verification for faster loan approvals."
  - "Accelerate primacy for high-potential customers."

**Body Copy:**
- Sentence case for body text
- 2-3 sentence paragraphs
- Use bullet points for features/benefits
- Include specific numbers when possible

**CTAs:**
- Primary: "Contact Sales," "Get Started," "Learn More"
- Secondary: "View docs," "See Integrations"
- Always action-oriented

## Customer Story Format

### Structure
1. **Headline:** "[Company] + Impact Statement"
   - "Compass Mortgage Finds a Better Way to Save 60-80% on Verifications"
   - "HFS Reduces Income Fraud, Improves Team Efficiency by 10-15%"

2. **Key Metrics** (hero stats with large numbers)
   - +60-80% savings on verifications
   - 65% conversion rate
   - +10-15% improvement in efficiency

3. **Problem Statement**
   - What challenge did they face?
   - Why did they need a solution?

4. **Truv Solution**
   - Which products did they implement?

5. **Quote-Led Narrative**
   - Customer quotes throughout
   - Format: Name, Title, Company

6. **Results & Outcomes**
   - Specific metrics and improvements
   - Business impact

### Quote Style
- Use quotation marks, not italics
- Include attribution: Name, Title, Company
- Focus on outcomes and partnership
- Examples:
  - "Truv has given us the ability to lower costs, all while speeding up the verification process."
  - "There's no excuse for lenders to not look at a product like Truv. It works, it's easy for the borrower, the data set is superior."
  - "You get a partner dedicated to speed, data accuracy, product flexibility, competitive costs, and conversion."

### Customer Quote Themes
- Partnership/collaboration
- Cost savings
- Better data quality
- Improved borrower/user experience
- Operational efficiency

## Email Communication

### Subject Lines
- Lead with value/benefit
- Include numbers when possible
- Maximum 50 characters
- Lowercase, informal style
- No punctuation at end
- Examples:
  - "save 80% on income verifications"
  - "new: Truv + Encompass integration"

### Email Structure
1. **Paragraph 1:** REASON + PROBLEM (2 sentences)
2. **Paragraph 2:** VALUE with social proof (2-3 sentences)
3. **Paragraph 3:** CTA (1 sentence)

### Spacing
- Blank line between each paragraph
- No greeting at the start
- No signature block in templates (added by sender)

## Content Formatting

### Headings
Use clear hierarchy:
- **H1** - Page titles
- **H2** - Section headers
- **H3** - Subsection headers
- **H4** - Detail headers

### Numbers and Metrics
- Bold key numbers: **80%**, **$350**, **30 seconds**
- Use tables for comparing metrics
- Lead with the number when possible

### Lists
- Use bullet points for features/benefits
- Use numbered lists for sequences/steps
- Keep items parallel in structure

### Quotes
- Use blockquote formatting
- Always attribute with name, title, company
- Pull impactful customer quotes prominently

## Design Patterns

### Visual Style
- **Modern and minimalist** with clean lines
- **Curved transitions** for visual flow
- **High white space** for readability
- **Data-forward** - let numbers and metrics speak

### Interactive Elements
- Hover states: opacity changes and subtle color shifts
- Transitions: 0.3s-0.4s smooth ease timing
- Accordions: animated arrow rotations
- Focus states for accessibility

### Layout
- Mobile-first responsive design
- Primary breakpoint at 640px
- Consistent padding and margins
- Clear visual hierarchy
