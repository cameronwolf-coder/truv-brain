🎯 Event Dashboard & Automation System — Live
Built a full-stack event operations system this week. When Melissa creates a new event in HubSpot's custom Event objects, the entire task pipeline fires automatically — no manual setup needed.
Dashboard: truv-brain.vercel.app/hub

Event Task Automation Pipeline

A 4-step Pipedream workflow that triggers when a new Event is created in HubSpot:

Step Action
1. HubSpot Trigger New Event custom object created (webinar, conference, etc.)
2. Notion Playbook Fetches the Webinar Event Marketing Playbook (~31 tasks covering landing page creation through post-event follow-up)
3. HubSpot Tasks Auto-creates each playbook task with the right owner (Cam, Melissa, or Kaytren), due date (calculated relative to event date), and priority
4. Linear Issues Mirrors every HubSpot task as a Linear issue under a [MKTG-EVENT] project with matching assignee, due date, and priority

Bi-Directional HubSpot ↔ Linear Status Sync

Three sync directions keep both platforms in lockstep. The team works wherever they prefer — HubSpot or Linear — and everything stays in sync.

Direction Mechanism Latency
HubSpot → Linear (task creation) Pipedream step 4 after webinar task creation Immediate
Linear → HubSpot (status sync) Team-scoped Linear webhook → Pipedream → HubSpot task update ~5s
HubSpot → Linear (status sync) HubSpot workflow → Pipedream → Linear issue update ~10s

Loop prevention built in: both sync directions check current status before writing. If status already matches, the workflow exits early — no infinite update cycles. Verified in production.

Universal Event Form Processor (MKTG-943)

Built a single HubSpot workflow that handles all event form submissions automatically. When someone submits the "Upcoming Event Form," the workflow reads their utm_campaign value, searches Event custom objects for a match, and creates a contact ↔ event association. No per-event workflow needed.

Step Action
1 Set "Last Upcoming Event Form Submission Date" on contact
2 Set lifecycle stage to MQL
3 Send Slack notification to #webinar-signups
4 Custom code: read utm_campaign → search Event objects → create contact ↔ event association (type 40)

UTM-Based Event Tagging (MKTG-945)

Added utm_campaign as a property on Event custom object records, visible in the "About this Event" sidebar card. This is the key that links form submissions to the right event. To onboard a new event, just set its utm_campaign value to match the UTM used in the form's landing page — the Universal Event Form Processor handles the rest. First event configured: March Insider Public Sector (insider-gov-webinar).
