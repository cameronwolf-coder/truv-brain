HubSpot Events

Automated sync pipeline — every event created in HubSpot instantly becomes a trackable project in Linear.

Team: Growth Marketing Built: March 2026 Stack: Pipedream + GraphQL
The Problem

Manual event tracking across two systems

✗

Duplicate Entry

Every event was manually created in both HubSpot and Linear — double the effort, double the risk of errors.

⚠

Stale Data

Date changes, status updates, and new events in HubSpot were never reflected in the Linear timeline view.

⏱

No Single Source of Truth

Sales entered events in HubSpot, Marketing tracked them in Linear — neither system stayed in sync.

Architecture

Three-step webhook pipeline

1

HubSpot Workflow Triggers HubSpot

Event created or updated → fires webhook POST to Pipedream endpoint

2

Fetch Event Details Pipedream

Retrieves full event record from HubSpot CRM API — name, dates, location, status, type

3

Sync to Linear Linear

Search for existing project → create or update via GraphQL with mapped fields & status

What Was Built

Three components, fully automated

⚙

HubSpot Workflow

  • Triggers on Event create & property change
  • Re-enrollment enabled for updates
  • Custom object type 2-56342751
  • Webhook POST action to Pipedream
⚡

Pipedream Workflow

  • HTTP webhook trigger
  • Code step: fetch from HubSpot API
  • Code step: search + create/update Linear
📊

Linear Project Sync

  • [MKTG-EVENT] prefix for timeline views
  • Status mapping via UUID status IDs
  • Dedup by project name search
  • Dates, location, type in description
Field Mapping

HubSpot → Linear translation

HubSpot Event Linear Project
event_name → name with [MKTG-EVENT] prefix
event_start_date → startDate
event_end_date → targetDate
event_location → Included in description
event_url → Included in description
event_status → statusId (UUID mapping)
event_type → Included in description
Status Mapping

Lifecycle states stay in sync

New event Upcoming → Planned
Active In Progress → In Progress
Done Completed → Completed

Status IDs are mapped to Linear's project workflow UUIDs, ensuring updates propagate correctly for timeline and filter views.

The Result

Fully automated, zero manual entry

HubSpot is the single source of truth for events. Linear's timeline view updates automatically — no more copy-paste, no more stale data.
